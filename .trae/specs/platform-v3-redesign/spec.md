# 燃渡AI平台 v3.0 大版本升级规格文档

> **版本**: 3.0  
> **创建日期**: 2026-07-01  
> **目标**: 完成平台第三次大版本更新，实现微信扫码登录、全站UI重构、合规改造、功能增强

## Why

当前平台已完成了 v1.0（基础平台搭建）、v2.0（框架优化+功能增强）的迭代，但仍存在以下核心问题：

1. **登录体系单一**：仅支持邮箱/手机号注册登录，缺少国内用户习惯的微信扫码登录，用户注册转化率低
2. **UI 设计缺乏品牌辨识度**：当前 UI 虽功能完整，但与同类 AI 平台（豆包、扣子、通义千问等）同质化严重，缺乏大厂生产级视觉质量
3. **用户引导缺失**：新用户进入后无清晰的上手路径，AI 学员和工作流用户的需求混杂，缺乏分层引导
4. **合规风险**：平台未完善用户协议、隐私政策，未接入内容安全机制，不符合网络安全法和公安备案要求
5. **小程序数据孤岛**：微信小程序和网站使用独立账号体系，用户无法跨端使用，数据不互通

本次 v3.0 以"**登录升级 + UI 重构 + 合规改造 + 用户分层**"为核心目标，在现有硬件条件（阿里云 2 核 4G）下实现大厂级的产品体验。

### 竞品设计共同规律

1. **首页三范式**：豆包/ChatGPT（首页即对话）vs 扣子/Dify（Marketing+演示）vs 火山引擎（企业级 SaaS）
2. **视觉差异化**：Claude 米色+衬线、Kimi 极简黑白——避开蓝紫同质化
3. **功能区隔明显**：导航层级少（≤3层）、功能入口卡片化、信息密度适中
4. **NEW 标签驱动**：新能力用橙色小标签标记，引导用户探索
5. **数据背书**：大字号展示用户数/生成量/工作流数，增强信任
6. **动词化命名**：Dify 的 BUILD/CONNECT/START 模式，避免名词堆砌
7. **冷启动引导**：ChatGPT/豆包式的"建议提问"气泡，降低用户焦虑

### 燃渡 AI 的差异化定位

- **定位**：面向中文用户的 AI 学习 + 工作流一站式平台（区别于豆包的纯对话、扣子的纯开发平台）
- **差异化**：学院（学习路径）+ 工作台（即用工具）+ 市场（模板复用），三者合一
- **视觉方向**：参考 Claude 的米色温暖系 + Kimi 的极简留白 + Dify 的叙事流，避免蓝紫渐变

## What Changes

### 模块一：微信扫码登录体系（P0 紧急）

**改造背景**：当前平台仅支持邮箱/手机号登录，国内用户更习惯微信扫码登录。微信扫码登录可显著降低注册门槛，同时为后续小程序数据互通打下基础。

- 接入微信开放平台网站应用扫码登录（OAuth 2.0）
- 实现微信账号与现有邮箱/手机号账号的绑定与解绑
- 统一用户身份标识（UnionID），支持同一主体下微信生态（网站+小程序）数据互通
- 老用户引导绑定微信账号

### 模块二：全站 UI 重构（P0 高优先）

**改造背景**：当前 UI 与同类 AI 平台高度同质化（蓝紫渐变+卡片墙），缺乏品牌辨识度，需要达到大厂生产级视觉质量。

- 首页重构：Hero 区 + 叙事流 + 数据背书 + 冷启动引导
- 登录/注册页重构：微信扫码入口 + 邮箱手机号双注册 + 品牌一致性
- 统一设计语言：米色温暖系主色调 + 极简留白 + 动词化命名
- 新增用户引导流程（Onboarding）：新用户首次登录 → 选择身份（学员/效率用户） → 定制首页推荐
- 全站交互动效优化：滚动揭示、按钮反馈、状态转换

### 模块三：合规与安全改造（P0 紧急）

**改造背景**：平台尚未完善法律合规机制，存在公安备案缺失、隐私政策不完整、内容安全机制缺失等合规风险。

- 完善《用户协议》和《隐私政策》页面（满足《个人信息保护法》要求）
- 内容安全机制：AI 生成内容敏感词过滤（接入阿里云内容安全 API）
- 完善用户注册实名制（手机号实名 + 微信 UnionID 关联）
- 检查公安联网备案状态（已完成备案，需确认备案号展示）
- 敏感操作日志记录（满足《网络安全法》要求）

### 模块四：学习路径与用户分层（P1 高优先）

**改造背景**：AI 学员和工作流用户的需求不同，需要在产品和内容层面做分层引导。

- 新增"学习路径"功能（燃渡学院重构）：按职业/场景设计的 AI 学习路线
- 用户画像选择（Onboarding）：首次登录让用户选择"学习 AI"或"使用工具"，定制首页内容
- 工作流分类重构：按行业（电商/新媒体/教育/企业）和场景（内容生成/数据分析/自动化）分类
- 学习进度追踪：记录用户观看教程、完成练习的进度，增强粘性

### 模块五：功能增强（P1 中优先）

- 智能体对话增强：多 Bot 切换、收藏提示词、对话导出
- 工作流增强：工作流收藏/评分/评论、数据统计（使用次数/成功率）
- 用户中心增强：使用数据中心（消耗排行、功能热度）、积分体系展示

## ADDED Requirements

### Requirement: 微信扫码登录

系统 SHALL 提供微信开放平台扫码登录功能，用户可通过微信扫码完成账号登录或注册。

#### Scenario: 新用户微信扫码登录
- **WHEN** 用户点击"微信登录"按钮，用微信扫描二维码
- **THEN** 若微信账号未关联现有账号，系统创建新账号并登录，跳转首页或引导页
- **AND** 返回 Set-Cookie httpOnly token（同现有登录逻辑）

#### Scenario: 老用户绑定微信
- **WHEN** 已登录用户访问"账号设置 → 绑定微信"
- **THEN** 显示微信二维码，用户扫码后账号成功绑定 UnionID
- **AND** 绑定后可通过微信扫码直接登录

#### Scenario: 微信登录与手机号登录共存
- **WHEN** 用户使用微信登录后，又使用手机号登录同一账号
- **THEN** 系统识别同一 UnionID，将两个登录方式关联到同一用户账号
- **AND** 用户可通过任一方式登录

### Requirement: 用户引导流程（Onboarding）

系统 SHALL 在新用户首次登录后展示用户引导流程，帮助用户选择身份和定制首页内容。

#### Scenario: 新用户首次登录
- **WHEN** 用户首次登录（无论微信还是手机号）
- **THEN** 展示 Onboarding 页面，让用户选择"我想学 AI"或"我想用工具"
- **AND** 根据选择，定制首页推荐内容（学院优先 vs 工作台优先）

### Requirement: 用户协议与隐私政策

系统 SHALL 提供《用户协议》和《隐私政策》页面，满足《个人信息保护法》要求。

#### Scenario: 访问合规页面
- **WHEN** 用户访问 /privacy 或 /terms
- **THEN** 展示完整的隐私政策和用户协议，包含数据收集范围、使用目的、用户权利等内容
- **AND** 登录/注册页面显示"登录即表示同意《用户协议》和《隐私政策》"

### Requirement: 内容安全过滤

系统 SHALL 对 AI 生成的内容进行敏感词过滤，调用阿里云绿网 API 检测。

#### Scenario: AI 生成内容包含敏感词
- **WHEN** AI 返回的内容触发阿里云绿网内容安全检测
- **THEN** 替换敏感词为 ***，或返回"该内容暂不可展示"提示
- **AND** 记录过滤日志（不展示给用户）

### Requirement: 首页叙事流重构

系统 SHALL 重构首页为叙事流结构，包含 Hero 区、数据背书、功能展示、用户评价和 CTA。

#### Scenario: 首页滚动叙事
- **WHEN** 用户访问首页并滚动
- **THEN** 每个 section 进入视口时触发 stagger 交错淡入动画
- **AND** 叙事顺序为：品牌承诺 → 数据背书 → 功能分类 → 学员评价 → 定价预览 → 行动召唤

### Requirement: 学习路径功能

系统 SHALL 提供按职业/场景设计的 AI 学习路径，帮助用户系统化学习 AI。

#### Scenario: 浏览学习路径
- **WHEN** 用户访问学院页面
- **THEN** 展示预设的学习路径（如"电商运营 7 天入门"、"新媒体内容创作"）
- **AND** 每条路径包含目标说明、教程列表、预计完成时间
