## 一、现状问题诊断

### 1.1 当前平台问题

| 问题类别 | 具体表现 | 影响程度 |
|---------|---------|---------|
| **布局结构** | 信息密度不当，核心功能藏得过深 | ⭐⭐⭐⭐⭐ |
| **视觉层次** | 标题/正文/辅助文字层级混乱 | ⭐⭐⭐⭐ |
| **组件质量** | 按钮/卡片/输入框交互反馈粗糙 | ⭐⭐⭐⭐ |
| **品牌辨识** | 与同类AI平台（蓝紫渐变）同质化 | ⭐⭐⭐⭐ |
| **动效体验** | 生硬的过渡，缺少micro-interaction | ⭐⭐⭐ |
| **移动端** | 响应式适配不完整，触摸目标过小 | ⭐⭐⭐⭐ |
| **内容组织** | 功能入口分散，用户路径不清晰 | ⭐⭐⭐⭐⭐ |

### 1.2 用户体验痛点

```
用户进入首页 → 不知道这个平台能做什么 → 找不到想用的功能 → 离开

原因分析：
1. Hero区没有清晰传达核心价值主张
2. 功能入口过多且分散
3. 缺乏用户角色引导（学员 vs 效率用户）
4. 注册/登录门槛高，微信扫码未作为首选
```

### 3.2 导航重构

**当前导航问题：**
```
探索(首页) / 创作(工作台) / 市场(工作流) / 学院(教程) / 智能体 / 定价 / 用户中心
```

| 定价 | **定价** | 保留 |

**视觉规范：**
```
主按钮：
- 背景：--color-primary (渐变可选)
- 文字：白色
- 圆角：--radius-full (胶囊) 或 --radius-md
- padding: 12px 24px (lg) / 10px 20px (md) / 8px 16px (sm)
- hover: 上浮2px + 阴影加深
- active: 下沉1px
- disabled: opacity 0.5, cursor not-allowed
- loading: 显示spinner, 文字变淡
```
