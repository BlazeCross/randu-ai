## 一、整体架构

### 1.1 微信生态账号体系

```
微信用户 (openid) 
     ↓ 微信开放平台绑定同一主体
微信小程序 (unionid)
     ↓ 同一微信开放平台账号
网站应用 (unionid)
```

微信生态中，同一微信开放平台账号下的多个应用（网站+小程序）共享同一个 `unionid`，这是实现网站和小程序账号互通的核心。

### 1.2 登录流程

```
┌─────────────┐    1. 扫码授权     ┌─────────────┐
│  用户浏览器  │ ←───────────────→ │  微信二维码  │
└──────┬──────┘    2. 用户确认     └──────┬──────┘
       │                                  │
       │ 3. 回调 code                     │
       ├────────────────────────────────→ │ 回调到网站后端
       ↓                                  ↓
┌──────────────────────────────────────────────────┐
│              燃渡AI 网站后端                       │
│  4. code → 换 openid/access_token                  │
│  5. openid → 查询/创建用户                        │
│  6. 签发 JWT，返回登录成功                        │
└──────────────────────────────────────────────────┘
```

### 1.3 账号绑定流程

```
┌─────────────┐    绑定微信扫码     ┌─────────────┐
│  已登录用户  │ ←───────────────→ │  微信二维码  │
└──────┬──────┘                   └──────┬──────┘
       │ 3. 回调 code                  │
       ├────────────────────────────→ │
       ↓                               ↓
┌──────────────────────────────────────────────────┐
│              燃渡AI 网站后端                       │
│  4. code → 换 openid → 绑定到当前用户              │
│  5. 返回绑定成功                                  │
└──────────────────────────────────────────────────┘
```

## 二、前置条件

### 2.1 微信开放平台注册

1. 注册微信开放平台账号：https://open.weixin.qq.com/
2. 完成开发者资质认证（个人/企业，费用 ¥300/年）
3. 创建"网站应用"，填写信息：
   - 应用名称：燃渡AI
   - 应用描述：AI 工作流智能体平台
   - 网站地址：`https://www.randuai.cn`
   - 回调域：`https://www.randuai.cn`
4. 等待审核（1-3 个工作日）
5. 审核通过后获取 AppID 和 AppSecret

### 2.2 微信开放平台 vs 公众平台

| 平台 | 登录方式 | unionid 获取 | 适用场景 |
|------|----------|-------------|----------|
| 微信开放平台 | 网站应用扫码登录 | ✅ 支持 | PC 网站 |
| 微信公众平台 | 公众号内网页授权 | ✅ 支持 | 公众号 H5 |

**选择**：微信开放平台（网站应用扫码登录），扫码体验好，与小程序同主体。

### 2.3 小程序与网站的 unionid 关联

微信开放平台将同一主体的网站应用和小程序关联后，用户在任一端的 unionid 相同。

**条件**：
- 微信开放平台账号已认证
- 网站应用和小程序绑定在同一开放平台账号下
- 用户已关注公众号或使用过小程序（unionid 才会在两端一致）

## 三、API 设计

### 3.1 获取二维码参数

```
GET /api/auth/wechat/qrcode
```

**响应**：
```json
{
  "appid": "wx1234567890abcdef",
  "redirect_uri": "https%3A%2F%2Fwww.randuai.cn%2Fapi%2Fauth%2Fwechat%2Fcallback",
  "response_type": "code",
  "scope": "snsapi_login",
  "state": "random_csrf_token_abc123"
}
```

前端用这些参数生成微信二维码（使用微信 JS-SDK 或第三方库如 `miniprogram-login`）。

### 3.2 微信回调

```
GET /api/auth/wechat/callback?code=CODE&state=STATE
```

**处理逻辑**：
1. 验证 state（防 CSRF）
2. 用 code 换取 access_token 和 openid
3. 用 access_token 获取 unionid
4. 查询数据库是否有 unionid 匹配的账号
   - 有 → 直接登录，返回 JWT
   - 无 → 创建新账号，返回 JWT
5. 前端存储 token，跳转首页

**响应**（302 重定向到首页）：
```
Set-Cookie: token=JWT; HttpOnly; Secure; SameSite=Lax; Max-Age=604800
Location: https://www.randuai.cn/?wechat_login=success
```

### 3.3 绑定微信

```
POST /api/auth/wechat/bind
Authorization: Bearer <token>
Body: { "code": "微信扫码获取的code" }
```

**处理逻辑**：
1. 验证用户已登录
2. 用 code 换取 unionid
3. 查询 unionid 是否已被其他账号占用
   - 被占用 → 返回 409 冲突
   - 未占用 → 绑定成功

### 3.4 解绑微信

```
POST /api/auth/wechat/unbind
Authorization: Bearer <token>
```

## 四、数据库设计

### 4.1 User 模型扩展

```prisma
model User {
  id            String    @id @default(cuid())
  email         String?   @unique
  phone         String?   @unique
  passwordHash  String?
  
  // 微信相关（新增）
  wechatUnionId String?   @unique  // 微信 UnionID，用于跨端识别
  wechatOpenId  String?   // 微信 OpenID（每个应用不同）
  wechatBindAt  DateTime? // 绑定时间
  
  // ... 其他字段不变
}
```

### 4.2 UnionID 获取逻辑

微信返回的 unionid 字段：
- 如果用户曾在微信生态中（小程序/公众号）使用过，返回 unionid
- 新用户可能只有 openid，unionid 为空（这种情况不能跨端互通，但不影响登录）

## 五、前端实现

### 5.1 微信二维码生成

使用微信开放平台官方 `QRcode` 库或第三方封装库。

**推荐方案**：使用 `weixin-js-sdk` + 自定义二维码容器

```tsx
// components/auth/WechatQRCode.tsx
"use client";
import { useEffect, useState } from "react";

export function WechatQRCode({ redirectUri, onSuccess }: { 
  redirectUri: string;
  onSuccess: (code: string) => void;
}) {
  useEffect(() => {
    // 动态加载微信二维码生成脚本
    const script = document.createElement("script");
    script.src = "https://res.wx.qq.com/connect/zh_CN/htmledition/js/qrcode.js";
    document.body.appendChild(script);
    
    script.onload = () => {
      // 调用微信 API 生成二维码
      // 需要先从后端获取 appid 和其他参数
    };
  }, [redirectUri]);
  
  return <div id="wechat-qrcode" />;
}
```

### 5.3 微信登录回调处理

微信回调 URL 不返回页面内容，而是处理后重定向。回调页面需要处理 URL 参数：

```tsx
// 检查 URL 中的登录成功参数
useEffect(() => {
  const params = new URLSearchParams(window.location.search);
  if (params.get("wechat_login") === "success") {
    // 清除 URL 参数
    window.history.replaceState({}, "", "/");
    // 触发刷新（已通过 cookie 登录）
    window.location.href = "/";
  }
}, []);
```

## 六、安全考量

### 6.1 CSRF 防护

每个二维码生成时生成随机的 `state` 参数，存储在 Session 或 Redis 中。
回调时验证 state 是否匹配，防止 CSRF 攻击。

### 6.2 账号绑定安全

- 绑定微信需要用户已登录（JWT 验证）
- 绑定时需要二次验证（如输入密码）
- 解绑需要密码验证

### 6.3 unionid 泄露防护

- unionid 不返回给前端，只在后端存储
- 日志脱敏处理

## 七、与小程序账号互通

### 7.1 小程序扫码登录网站

小程序使用微信登录后，微信返回 unionid → 与网站数据库中的 unionid 匹配 → 直接登录。

### 7.2 网站账号登录小程序

网站用户绑定微信后，unionid 关联到用户。在小程序中：
- 微信登录 → 获取 unionid → 查询数据库 → 匹配网站账号 → 返回网站账号信息

### 7.3 数据同步

| 数据 | 网站 → 小程序 | 小程序 → 网站 |
|------|-------------|-------------|
| 账号信息（邮箱/手机） | ✅ | ✅ |
| 积分余额 | ✅ | ✅ |
| 订阅状态 | ✅ | ✅ |
| 对话历史 | ❌（本地） | ❌ |
| 学习进度 | ✅ | ✅ |
